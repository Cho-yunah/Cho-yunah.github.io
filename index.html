Git Basic

GUI 의 방식을 배우기전 CLI 의 방식을 먼저 배우는 이유, git이 어떠한 명령어의 원리로 움직이는지를 알기 위함이다.

git bash를 다운로드 하여 실행한 후, git의 버전을 알기위해 명령어 [git -version]를 입력한다.

현재 2020년 7월 기준 2.27.0 windows 버전이면 최신이다.

/* 하단부터는 명령어 입력을 따로 말하지 않고 [ ] 괄호로 대신한다. */

[git config] 최초 실행시 1회 입력하는 명령어

—> git config —globl user.name “your_name”////

—> git config —global user.email “your_email@example.com“

  • 잘 입력이 되었는지 확인하려면 [git config —list]를 입력하면 된다. 잘 못 입력했을 경우 위의 이름과 이메일 명령어를 다시 입력하면 된다.

[git init] 현재 디렉토리를 로컬저장소로 설정하는 명령어

—> 이 명령어 이후에 (master) 브랜치로 보이면 성공한 것이다.

—> init 명령어를 취소하려면 [rm -r .git]을 입력한다.

[ls] =⇒ 현재 디렉토리에 어떤 하위 디렉토리가 있는지 알수 있는 명령어 (또는 dir)

[pwd] =⇒ 현재 위치한 곳을 알수 있는 명령어 (/c/Users/조윤아)

[mkdir 폴더이름] =⇒ (이름) 폴더 만들수 있는 명령어

—> 폴더를 새롭게 만든 후에는 [ls]명령어로 제대로 추가가 되었는지 확인하는 것이 좋다.

—> 만들고 싶은 디렉토리 사이에 띄어쓰기를 넣고 연속해서 씀으로써 ..  명령어 한번으로 여러개의 디렉토리를 만들수 있다. ex) cd css js assets

[cd fds17] =⇒ (change diretory) 선택한 디렉토리로 이동시켜주는 명령어 (=내 위치 디렉토리 바꿔줘)

[cd ..] =⇒ 한단계 위의 디렉토리로 나가기 ///// [cd../..] =⇒ 두단계 위로 나가기

cd documents/dev=⇒ 폴더를 갈때 /로 두단계 바로 이어갈 수 있음

[touch] 빈 파일을 만들때 사용하는 명령어 (안에 내용이 없음)

[echo] 파일을 만들면서 그안에 내용을 넣을 때 사용하는 명령어

—> $ echo  '<!doctype html>' >> index.html  (index.html 이라는 파일안에 <!doctype html>이라는 태그를 삽인한다)

[cat 파일이름] 터미널 안의 모든 텍스트(내용)을 확인할 수 있는 명령

[rm 파일이름] 파일 삭제 명령어( 논리적인 성격임 접근을 지운다는것 존재자체는 남아있음 접근이 불가)

[명령어1 && 명령어2] 한줄에 넣는 명령어가 다를때 &&를 중간에 사용한다.

—> $ cd assets && ls (assets 디렉토리를 찾은 후에 그 하위 디렉토리가 무엇인지 알려줘!)

—> 명령어가 같을 때는 스페이스를 사용하면 된다.

[ctrl c] 명령어를 잘못 실행했을 때 빠져나올 수 있는 명령어

[ls -a] 숨겨진 디렉토리까지 보여주는 명령어

—> 보통 .git/으로 끝나는 디렉토리가 날라가면 그간의 이력과 과정이 모두 없어지기 때문에, 업데이트를 하기가 매우 어려워진다. 그래서 이 중요한 폴더는 보통 숨김 처리된다.

[git status] 현재 상태를 알고 싶을 때 사용하는 명령어

[git add 파일이름 또는 .] 커밋할 파일을 준비영역(stage)로 옮긴다.

—> 파일이름을 넣으면 입력한 특정파일만 추가 // 

   . 을 넣으면 디렉터리 내 모든 파일을 추가

—> stage로 add한 파일을 워킹 디렉토리로 다시 옮길때는 

    [git rm —cached 파일이름 또는 .]

[git commit -m “커밋 메세지’] stage에 올려둔 파일을 로컬저장소에 저장하는 명령어

[git log] 커밋 이력을 상세 조회하는 명령어

—> [git log —oneline] 커밋 이력중에 커밋 ID와 타이틀 메세지만 조회

[git remote] 로컬 저장소와 원격 저장소를 연결한다.

—> [git remote add origin] github 원격 저장소와 연결한다.

—> [git remote -v] 연결된 원격 저장소를 확인한다.clea
